================================================================================
                      DESIGN JUSTIFICATION
================================================================================

WHY SUFFIX ARRAY OVER SUFFIX TREE:

Advantages:
-----------
1. Simpler Implementation
   - 200 lines vs 500+ for Ukkonen's algorithm
   - Fewer edge cases, easier to verify

2. Memory Efficiency
   - 2N integers vs 10N+ for suffix trees
   - ~800 bytes/base vs ~2000+ bytes/base

3. Cache Performance  
   - Sequential array access
   - Better CPU cache utilization

4. Practical Speed
   - O(|P| log |T|) is fast enough for genomics
   - <1ms search on 4.6M bases

5. Lower Risk
   - Easier to debug and test
   - More suitable for academic project

Trade-offs:
-----------
- Search: O(|P| log |T|) vs O(|P|) [Acceptable for our use case]
- Construction: O(N log N) vs O(N) [Can be optimized with SA-IS later]

Conclusion:
-----------
For this project's requirements (exact matching, repeat discovery, 
reasonable performance on bacterial genomes), Suffix Array + LCP 
is the OPTIMAL choice.

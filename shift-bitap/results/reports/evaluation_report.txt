KEY FINDINGS
================================================================================

1. LATENCY/TIME:
   - Search time scales linearly with text length (O(n))
   - Preprocessing time is negligible (<1ms for all patterns)
   - Throughput: Millions of base pairs per second

2. PREPROCESSING TIME:
   - Bitmask creation is extremely fast (<0.01ms typical)
   - Constant time O(m) for pattern length m
   - Independent of text length

3. MEMORY USAGE:
   - Constant space O(sigma) for alphabet size sigma
   - For DNA (sigma=4), memory footprint is minimal
   - No index structure needed unlike suffix arrays/trees

4. ACCURACY:
   - 100% accurate for exact matching
   - High precision/recall for approximate matching
   - Correctly handles edit distances up to pattern length

5. SCALABILITY:
   - Linear time O(n) confirmed across all tests
   - Performance consistent across pattern lengths (5-30bp)
   - Efficient for both small (48K bp) and large (4.6M bp) genomes

6. ROBUSTNESS:
   - Small alphabet (ACGT) ideally suited
   - Graceful degradation with increased edit distance
   - Suitable for patterns up to 64bp (word size limit)

================================================================================
CONCLUSION
================================================================================

The Shift-Or/Bitap algorithm demonstrates excellent performance for
DNA sequence matching with the following characteristics:

STRENGTHS:
  • Extremely fast preprocessing (<1ms)
  • Linear time search O(n)
  • Constant memory usage O(1) for DNA
  • Supports approximate matching efficiently
  • Simple implementation with bit-parallel operations

LIMITATIONS:
  • Pattern length limited to word size (typically 64bp)
  • Less efficient for very long patterns
  • Approximate matching cost increases with edit distance

RECOMMENDED USE CASES:
  • Short to medium patterns (5-50bp)
  • Small alphabets (DNA, RNA, protein)
  • Applications requiring low memory footprint
  • Real-time or streaming sequence analysis
  • Approximate matching with small edit distances (k ≤ 3)
